## Дизайн

### Характер продукта
Штурман — это надёжный, спокойный проводник. Не кричащий стартап,
не скучная госуслуга. Где-то между туристической картой и умным другом,
который разобрался в теме и объясняет просто.

### Цветовая палитра
```
--green-dark:   #1B4332   /* заголовки, шапка, акцент */
--green-mid:    #2D6A4F   /* кнопки, активные элементы */
--green-light:  #40916C   /* иконки, подчёркивания */
--green-pale:   #D8F3DC   /* фон пузырей агента, hover */
--green-xpale:  #F0FAF3   /* фон страницы */
--accent:       #52B788   /* прогресс-бар, выделения */
--text:         #1B1B1B   /* основной текст */
--text-muted:   #6B7280   /* подсказки, мета-текст */
--white:        #FFFFFF
```

### Типографика
- Шрифт системный: `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`
- Заголовок агента: 22px, weight 700
- Текст чата: 15px, line-height 1.55
- Подсказки: 13px, opacity 0.7
- Кнопки: 14px, weight 500

### Форма и радиусы
- Карточки и контейнеры: `border-radius: 16px`
- Кнопки-варианты: `border-radius: 24px` (таблетки)
- Пузыри агента: `border-radius: 4px 16px 16px 16px`
- Пузыри пользователя: `border-radius: 16px 4px 16px 16px`

### Анимации
- Появление сообщений: `fadeUp` 0.3s ease
- Переходы кнопок: 0.18s ease
- Прогресс-бар: 0.4s ease
- Индикатор печати: три точки, blink-анимация
- Без тяжёлых анимаций — всё должно быть быстрым

### Иконки и эмодзи
Используй эмодзи вместо иконочных библиотек — они универсальны
и работают везде без подключения:
- 🗺️ — логотип агента
- 🏡 ИЖС, 🌱 ЛПХ, 🌻 садоводство, 💼 бизнес
- ✅ да, ❌ нет, ⚠️ предупреждение, 🔄 начать сначала

## Адаптивная вёрстка

**Принцип: mobile-first.** Сначала делай для экрана 375px, потом расширяй.

### Контрольные точки
```css
/* Мобильный (по умолчанию): 320px–599px */
/* Планшет:  @media (min-width: 600px)  */
/* Десктоп:  @media (min-width: 900px)  */
```

### Обязательные правила адаптивности
- `max-width: 680px` для основного контейнера, центрировать на десктопе
- На мобильном — контейнер занимает `100vw`, `min-height: 100dvh`
- Кнопки-варианты: `flex-wrap: wrap`, минимальная ширина кнопки `fit-content`
- Шрифт в чате на мобиле: не меньше 15px (иначе iOS масштабирует)
- Поле ввода: `font-size: 16px` — чтобы iOS не зумил при фокусе
- Отступы на мобиле: `padding: 12px 16px` (не меньше 16px по бокам)
- Тач-цели: высота кнопок минимум 44px (стандарт Apple HIG)

### Что проверять после каждого изменения
1. Открыть DevTools → режим мобильного устройства (iPhone SE, 375px)
2. Убедиться что кнопки не обрезаются
3. Убедиться что текст не выходит за края
4. Проверить что поле ввода не перекрывает контент при открытии клавиатуры

## Стиль текста в интерфейсе и коде

### Тон агента
- Разговорный, тёплый, уверенный
- Обращение на «вы» (уважительно, но не официально)
- Называть пользователя по имени, если знаем
- Эмпатия в нужный момент: не лезть с утешением без повода,
  но в кризисный момент — сначала признать ситуацию, потом решение

### Что нельзя
- Канцелярит: «в соответствии с», «руководствуясь», «надлежащим образом»
- Длинные вводные: «Для того чтобы ответить на ваш вопрос, необходимо...»
- Слова-паразиты: «безусловно», «несомненно», «следует отметить»
- Пассивный залог там, где можно активный
- Перегруженные предложения — одна мысль = одно предложение

### Как правильно
```
❌  «Осуществление подачи заявления производится посредством...»
✅  «Подайте заявление через...»

❌  «В случае получения отказа рекомендуется произвести анализ...»
✅  «Получили отказ? Вот что делать дальше.»

❌  «Данный шаг является обязательным условием для...»
✅  «Этот шаг нужен, потому что без него администрация откажет.»
```

### Стиль сообщений агента
- Первое сообщение: приветствие + одна понятная цель
- Вопрос роутера: жирный текст вопроса + курсивная подсказка под ним
- Предупреждение (⚠️): коротко, без запугивания, с выходом
- Результат: сначала объяснение почему, потом карточка стратегии
- Конец сессии: конкретное следующее действие или вопрос

## Технические правила

### JavaScript
- Без фреймворков на фронте пока не нужны — чистый JS
- Имена переменных и функций: camelCase, по-английски
- Комментарии в коде: по-русски, коротко — объясняют «почему», не «что»
- Никаких `var` — только `const` и `let`

### CSS
- Переменные через `--custom-properties` в `:root`
- Без `!important` — если нужно, значит проблема в специфичности
- Мобильные стили — базовые, десктоп — через `@media (min-width: ...)`

### HTML
- `lang="ru"` на `<html>`
- `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- Семантические теги: `<header>`, `<main>`, `<footer>`, `<button>`, `<input>`
- Alt-тексты для изображений (если появятся)

### Производительность
- Никаких тяжёлых библиотек без явной необходимости
- Шрифты — системный стек, не Google Fonts
- Изображения — только если реально нужны

## Как объяснять пользователю что делаешь

Пользователь — новичок в разработке. Перед каждым действием
объясни простым языком:

- **Что создаём:** название файла и его роль
- **Зачем:** какую проблему решает
- **Что увидишь в результате:** конкретно, в браузере или терминале

После создания — одно предложение: что получилось и что дальше.

Не используй термины без объяснения. Если термин неизбежен —
объясни в скобках: «роутер (это код, который решает какую страницу показать)».
